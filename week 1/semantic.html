1. Introduction

What Are Semantics?

Semantics is the study of the meanings of words and phrases in a language.

Semantic elements = elements with a meaning.

What Are HTML5 Semantic Elements?

A semantic element clearly describes its meaning to both the browser and the developer.

Examples of non-semantic elements: <div> and <span> - Tells nothing about its content.

Examples of semantic elements: <article>, <section>, and <aside> - Clearly defines its content.

2. Header

The <header> element specifies a header for a document or section.
The <header> element should be used as container for introductory content and multiple <header> elements can be used throughout your web page!

Header Element vs. Heading Tags

By now, I more than sure you know about the varying heading tags. The heading tags range from <h1>-<h6>. These tags are good to use for topic headings throughout a webpage for emphasis. The header element is a block level element that can contain other elements contain introductory information for the web page. The header element will more than likely contain heading tags.

3. Nav

The <nav> element is used for site navigation. Menu items can be created with this element and multiple elements of this type can be used throughout a webpage.

(*You may notice that towards the top of the page that a <nav> element has been utilized for easier navigation to topics!)

It is common to hav global navigation across the top of the webpage and local navigation in a sidebar.

4. Section

A section semantic element is defined as a thematic grouping of content, usually with a heading. Section blocks of code normally will not make sense on their own if extracted from a web page.

5. Article

The <article> element specifies independent, self-contained content. An article should make sense on its own, and it should be possible to independently read from the rest of the site.

The section of wording above is an example of an article. It as well as the rest of the numbered topics can stand alone on the Internet.

Article vs. Section

Article and section elements are similar in nature in that both are block level elements used for sectioning content. Both are highly useful and offer even more information to the browser for context, but the major difference between these two elements is that an article can stand alone and a section can not stand alone.

6. Aside

The aside elemenet is intended for content that is not part of the flow of the text as it appears. The information in the aside element will still be related in some way.

This is an exmaple of an aside. Some type of terminology explanation or related content can be placed inside the <aside> element like a sidebar.

7. Footer

A footer typically contains the author of the document, copyright information, links to terms of use, contact information, etc.
You may have several <footer> elements in one document.
